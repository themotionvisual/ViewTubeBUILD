# VT_E1 Visual FX Studio

Use this skill when:
- Adding or modifying any effect in Unified FX Studio.
- Building effect panels, parameter schemas, live previews, or overlay clip creation.
- Integrating Remotion visual effects (noise, glitch, leaks, starburst, shader-like visuals).

## Core Contract
Implement effects via a registry-driven model:
- `EffectDefinition`
- `EffectParamDefinition`
- `EffectSessionState`
- `OverlayEffectPayload`

One active effect editor at a time. Live preview always reflects current active effect.

## Workflow
1. **Register effect** with id/label/category/defaults/param ranges.
2. **Bind preview renderer** to active session values.
3. **Expose target mode** (`selected-clip` or `new-overlay-clip`) with clip-first default.
4. **Implement apply mapper** for:
   - effect stack update on selected clip,
   - transparent overlay clip creation.
5. **Validate timeline behavior** (move/resize/reorder overlay clips).

## UI Rules
- Keep parameter controls consistent across effects.
- Show concise "what this changes" summary.
- Use one unified action row (apply/create).
- Avoid duplicate buttons and mixed terminology.

## Technical Rules
- Deterministic seeded randomness only.
- No heavy filter work outside active preview needs.
- Effect defaults should be visible and meaningful.
- Max settings should produce clearly stronger output without clipping artifacts.

## Validation Checklist
- Live preview updates on every parameter change.
- Switching effect resets visible controls to active effect only.
- Overlay clips render with transparent payload and expected blend mode.
- Saved presets restore exact values.
- Fixed seed yields reproducible frame output.
